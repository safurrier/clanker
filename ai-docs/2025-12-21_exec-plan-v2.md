# ExecPlan v2: Clanker9000 Gap Closure + Enhancements

This plan targets missing items identified against `clanker.md` and aligns with the v1 architecture already implemented. It is formatted to mirror the original ExecPlan structure while focusing on feature gaps and stability.

---

## 0. Scope Summary

Primary goals:
- Close remaining v1 design doc gaps (threaded chat, persona YAML, policy/profanity filter, admin commands, logging/metrics, voice ingest wiring).
- Strengthen Discord host behavior and tests.
- Keep SDK pure, testable, and free of Discord/network dependencies.
- Preserve opt‑in network smoke tests and avoid network calls in default CI.

## 2. Work Plan (Runnable Steps)

### Step 1 — RED: Threaded `/chat`
- Add tests validating thread creation and response posting (dpytest fakes or unit tests around handler logic).
- Implement thread creation in `src/clanker_bot/commands.py`.
- Command: `make check`

### Step 2 — GREEN: Persona YAML + provider config
- Add YAML loader in SDK (pure Python, no Discord imports) under `src/clanker/config/`.
- Add tests for parsing and validation (missing fields, invalid provider, default persona).
- Wire host to load persona + provider config at startup in `src/clanker_bot/main.py`.
- Command: `make check`

### Step 3 — GREEN: Policy + profanity filter
- Add a minimal policy implementation in `src/clanker/policies/`.
- Policy should validate prompt + context, return friendly errors.
- Add tests for policy behavior and integration with `respond()`.
- Command: `make check`

### Step 4 — GREEN: Shitpost image generation
- Wire ImageGen into `render_shitpost()` or the command handler to produce memegen images when a template requires it.
- Add tests for text + image flow (mock ImageGen).
- Command: `make check`

### Step 5 — GREEN: Voice ingest wiring (Discord host)
- Implement a host‑level voice receiver that feeds PCM into `clanker.voice.*` pipeline.
- Use a feature flag to enable voice receive, return a friendly message if unavailable.
- Add tests for BUSY state and that voice ingest is invoked when enabled.
- Command: `make check`

### Step 6 — GREEN: Admin commands + observability
- Add minimal admin commands (list active voice sessions, stop/start new meetings).
- Add structured logging for Context (sanitized) and counters for errors.
- Add tests for admin commands (dpytest or handler unit tests).
- Command: `make check`

### Step 7 — Network smoke tests (opt‑in)
- Add/extend network smoke tests for:
  - OpenAI LLM
  - OpenAI STT
  - Memegen image generation
- Keep gating via env vars; document usage in `README.md`.
- Command: `uv run --with pytest pytest -q -m network`

## 3. Validation & Acceptance

- Default CI:
  - `make check`
- Network smoke (opt‑in):
  - `uv run --with pytest pytest -q -m network`
- Manual smoke:
  - `scripts/smoke_discord_openai.py`

PASS criteria:
- Threaded `/chat` and `/speak` confirm thread creation + response placement.
- Persona YAML loads cleanly with validation.
- Profanity policy denies clearly bad input with friendly error.
- Shitpost can produce both text and image output.
- Voice ingest is either wired or returns a friendly “feature unavailable” message.
- Admin commands work and are gated by admin list.
